# AGENTS.md

Lumen project — image/video generation with `.lumen` files and one pipeline server on Modal.

## Structure

| Path                               | Purpose                                     |
| :--------------------------------- | :------------------------------------------ |
| server/                            | Pipeline server deployed on Modal           |
| server/pipelines/                  | Pipeline modules (auto-discovered)          |
| server/AGENTS.md                   | Server docs (auto-generated by `lumen-sdk`) |
| server/lumen.log                   | Server log output (gitignored)              |
| \*.lumen                           | Generation config files (opened in VS Code) |
| .agents/skills/lumen-file/SKILL.md | How to author `.lumen` files                |

## Rules

- Python via uv for the server. Never pip install directly.
- `unknown` with type guards, not `any`. Explicit null handling, not `!`.
- Comment non-obvious intent only.
- To create or edit `.lumen` files, follow `.agents/skills/lumen-file/SKILL.md`.

## Workflow

1. Write pipelines in `server/pipelines/` — see `server/AGENTS.md`
2. Create `.lumen` files to configure generation params for that server (see `.agents/skills/lumen-file/SKILL.md`)
3. The VS Code extension manages the dev server and renders `.lumen` files
4. Check `server/lumen.log` for server output and errors

## Quick reference

```sh
cd server
uv run pytest            # validate pipeline contracts
uv run ruff check .      # lint
lumen-sdk sync           # update framework docs and skills
```

## Prerequisites

Run `bash doctor.sh` to check prerequisites and bootstrap the server environment.
